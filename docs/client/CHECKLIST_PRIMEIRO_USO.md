# Smart Tech PDV — Checklist de Primeiro Uso

Use este checklist na primeira instalação do cliente.

## Empresa e usuário

- [ ] Empresa cadastrada.
- [ ] Telefone e dados principais conferidos.
- [ ] Usuário administrador configurado.
- [ ] Senha padrão alterada quando aplicável.
- [ ] Acesso testado após fechar e abrir o sistema.

## Licença

- [ ] Licença ativada.
- [ ] Validade conferida.
- [ ] `device_id` conferido no diagnóstico, sem copiar valor completo.
- [ ] Sistema abre offline após ativação.

## Impressão

- [ ] Impressora padrão selecionada.
- [ ] Teste 58mm realizado, se usado.
- [ ] Teste 80mm realizado, se usado.
- [ ] Teste A4/PDF realizado, se usado.
- [ ] Acentuação conferida.
- [ ] Corte de papel conferido.
- [ ] QR/logo conferidos quando usados.

## Dados iniciais

- [ ] Cliente de teste criado.
- [ ] Produto de teste criado.
- [ ] Venda de teste criada.
- [ ] OS de teste criada.
- [ ] Cobrança de teste criada.
- [ ] Financeiro e Fluxo de Caixa conferidos.

## Persistência

- [ ] Sistema fechado pelo X.
- [ ] Sistema aberto novamente.
- [ ] Dados de teste continuam aparecendo.
- [ ] Computador reiniciado, se possível.
- [ ] Sistema aberto novamente e dados conferidos.

## Backup

- [ ] Backup inicial criado.
- [ ] Local do backup identificado.
- [ ] Cliente orientado a fazer backup antes de update ou restore.

## Atualização

- [ ] Canal de atualização conferido.
- [ ] Cliente orientado a não apagar AppData.
- [ ] Cliente orientado a chamar suporte se a atualização falhar.
